# 🚀 Быстрый старт

### 1. Требования
Убедитесь, что на целевом сервере установлен [Node.js](https://nodejs.org) версии 16 или выше.

### 2. Клонирование и установка
Склонируйте репозиторий на сервер и установите все необходимые зависимости:

```bash
git clone https://github.com
cd dynastio-leaderboard
npm install
```

### 3. Первый запуск и инициализация
Запустите сервер стандартной командой для прохождения интерактивной настройки:

```bash
node index.js
```

> 💡 **Важно:** При самом первом старте скрипт обнаружит отсутствие файла `.env` и запустит интерактивный опрос в консоли. Вы сможете указать порт, выбрать тип базы данных (SQLite / JSON), задать кастомные пути и ввести секретный эндпоинт для отправки рекордов. Файл `.env` сгенерируется автоматически.

## 🎮 Подключение сервера к игре Dynast.io

Чтобы игровой сервер Dynast.io начал отправлять статистику и рекорды в ваш лидерборд, вам необходимо привязать его через конфигурационные файлы игры:

1. Перейдите в директорию с файлами вашего игрового сервера Dynast.io.
2. Найдите и откройте для редактирования файл конфигурации **`server.json`**.
3. Найдите параметр **`announcement_url_mirror`** и укажите в нём полный URL-адрес вашего развернутого веб-сервера, включая секретный путь, заданный в `.env` (например, `/take_records_secret`):

```json
{
  "announcement_url_mirror": "http://IP_ВАШЕГО_СЕРВЕРА:ПОРТ/take_records_secret"
}
```

4. Сохраните изменения в файле `server.json` и перезапустите игровой сервер. Теперь статистика игроков будет автоматически отправляться в базу данных лидерборда.

> ⚠️ **ВАЖНО:** Никому и никогда не сообщайте ваш секретный URL-путь (заданный вместо `/take`)! Если посторонние люди узнают этот эндпоинт, они смогут отправлять поддельные запросы со сторонних утилит (например, Postman) напрямую на ваш сервер, тем самым взламывая и накручивая любые значения в таблице лидеров.

## 📡 Документация API

### 1. Отправка рекордов
* **Метод:** `POST`
* **Путь:** Динамический (задается при инициализации, например: `/take_records_secret`)

### 2. Получение текущих топ-листов (День, Неделя, Месяц)
Возвращает отсортированные по убыванию очков списки за текущие периоды.
* **Метод:** `GET`
* **Путь:** `/leaderboard`

### 3. Запрос лидерборда за конкретную дату
Вы можете запросить архив за любой прошедший день. Поддерживаются ключевые слова (`day`, `week`, `month`) или точные даты в форматах `ДД-ММ-ГГГГ`, `ДД_ММ_ГГГГ`, `ДД.ММ.ГГГГ`.
* **Метод:** `GET`
* **Примеры путей:**
  * `/leaderboard/day` — за текущий день
  * `/leaderboard/18-06-2026` — за конкретный день
  * `/leaderboard/day=18.06.2026` — альтернативный формат даты

## 🛠️ Переменные окружения (.env)

Если вам потребуется вручную изменить конфигурацию без перезапуска мастера настроек, отредактируйте созданный файл `.env`:

* `PORT` — Порт, на котором разворачивается Express сервер.
* `NODE_ENV` — Режим работы (`development` или `production`).
* `DB_TYPE` — Используемый адаптер данных (`sqlite` или `json`).
* `DB_PATH` — Путь к файлу базы данных для SQLite адаптера.
* `JSON_DIR` — Имя папки, куда будут сохраняться таблицы в режиме JSON.
* `TAKE_ENDPOINT` — Секретный URL-путь для метода POST.

## 🔄 Настройка постоянной работы сервера (Запуск 24/7)

Чтобы игровой сервер продолжал работать в фоновом режиме после закрытия терминала и автоматически перезапускался при сбоях, выберите один из двух вариантов ниже.

> ⚠️ **Перед настройкой:** Обязательно запустите сервер вручную хотя бы один раз (`node index.js`), чтобы сгенерировать файл `.env` через мастер настроек!

### Вариант 1. Использование PM2 (Рекомендуемый, кроссплатформенный)

`PM2` — это специализированный менеджер процессов для Node.js, который предоставляет удобный мониторинг и управление логами.

1. Установите PM2 глобально на сервере:
   ```bash
   sudo npm install pm2 -g
   ```
2. Запустите игровой сервер под управлением PM2:
   ```bash
   pm2 start index.js --name "dynastio-leaderboard"
   ```
3. Настройте автоматический запуск PM2 при перезагрузке самого сервера (VPS):
   ```bash
   pm2 startup
   pm2 save
   ```

**Полезные команды для управления через PM2:**
* Посмотреть статус и потребление памяти: `pm2 status`
* Логи сервера в реальном времени: `pm2 logs dynastio-leaderboard`
* Перезапустить сервер: `pm2 restart dynastio-leaderboard`
* Остановить сервер: `pm2 stop dynastio-leaderboard`

### Вариант 2. Использование Systemd Service (Для Linux серверов)

Классический способ создания фонового демона средствами операционной системы Linux (Ubuntu/Debian).

1. Создайте файл конфигурации службы:
   ```bash
   sudo nano /etc/systemd/system/leaderboard.service
   ```
2. Вставьте в него следующее содержимое, заменив `/path/to/project` на **абсолютный путь** к папке с вашим скриптом `dynastio-leaderboard`, а `user` на имя вашего пользователя в Linux:
   ```ini
   [Unit]
   Description=Dynast.io Game Leaderboard API
   After=network.target

   [Service]
   Type=simple
   User=user
   WorkingDirectory=/path/to/project/dynastio-leaderboard
   ExecStart=/usr/bin/node index.js
   Restart=on-failure

   [Install]
   WantedBy=multi-user.target
   ```
3. Перезапустите менеджер конфигураций systemd, активируйте автозапуск службы при старте системы и запустите её прямо сейчас:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable leaderboard.service
   sudo systemctl start leaderboard.service
   ```

**Полезные команды для управления через Systemd:**
* Проверить статус работы: `sudo systemctl status leaderboard`
* Посмотреть логи работы приложения: `journalctl -u leaderboard.service -n 50 -f`
* Перезапустить службу: `sudo systemctl restart leaderboard`
* Остановить службу: `sudo systemctl stop leaderboard`
